# MoleOption 产品 PRD

## 1. 产品背景

现有加密杠杆产品通常在两个极端之间摇摆：

- 永续合约简单直接，但强平机制会在极端行情中放大用户损失体验。
- 期权风险有限，但定价、希腊值、权利金和流动性结构对普通用户过于复杂。

MoleOption 的产品目标是把“有限亏损”藏进协议内核，把“简单杠杆交易”留给用户界面。

## 2. 产品定位

MoleOption 是一个无传统爆仓的链上杠杆交易协议。用户存入固定本金后，可选择方向和杠杆开仓。该仓位最大亏损不超过本金，但盈利兑现取决于同市场对手方可兑现亏损。

## 3. 产品目标

### 3.1 首个真实资金版本目标

- 完成一个标的、一个抵押资产、一个杠杆倍数的生产级链上交易闭环。
- 用户可以开仓、查看仓位、触发或等待 epoch 结算、提交平仓请求并提取资金。
- 前端清楚展示参考盈亏、当前权益、最大亏损和盈利兑现风险。
- 合约从第一版即支持分片结算、结算队列和仓位懒结算，并通过关键不变量测试。

### 3.2 中期目标

- 支持多个杠杆隔离市场。
- 支持 BTC、ETH、SOL 等主流标的。
- 引入 keeper、索引服务、市场统计和风险仪表盘。
- 增加保险基金和多预言机防护。

### 3.3 长期目标

- 形成高流动性链上杠杆市场。
- 开放策略接口和生态集成。
- 引入治理、参数投票和协议收入分配机制。

## 4. 用户画像

### 4.1 普通杠杆交易者

需求：

- 快速判断方向并开仓。
- 不想学习期权术语。
- 希望知道最大可能亏损。
- 需要清晰的盈亏显示。

痛点：

- 害怕插针爆仓。
- 不了解资金费率和强平价格。
- 对链上复杂交互不熟悉。

### 4.2 高风险短线交易者

需求：

- 高杠杆。
- 低手续费。
- 零滑点或接近零滑点成交。
- 快速开平仓。

痛点：

- 传统高杠杆仓位容易被瞬时价格穿刺清算。
- 大额订单在订单簿中滑点明显。

### 4.3 稳健趋势交易者

需求：

- 5x-20x 中低杠杆。
- 明确风险上限。
- 不被高杠杆用户风险污染。

痛点：

- 不希望与 100x 用户共用资金池。
- 不希望短期波动打断长期趋势判断。

## 5. 核心用户故事

### 5.1 开仓

作为用户，我希望选择标的、方向、杠杆和保证金后开仓，并在确认页看到最大亏损、手续费、参考名义价值和该市场当前流动性状态。

验收标准：

- 用户必须确认“最大亏损为本金，盈利不保证全额兑现”。
- 开仓前展示预言机价格、价格更新时间、手续费、市场是否暂停。
- 开仓成功后生成仓位 ID。

### 5.2 查看仓位

作为用户，我希望看到我的仓位当前状态，包括参考盈亏、当前权益、已锁定损失、剩余本金、盈利兑现率和开仓参数。

验收标准：

- 参考盈亏与当前权益分开展示。
- 若当前权益为 0，界面显示“仓位仍存在，但当前无可提取权益”。
- 若市场处于暂停状态，界面解释原因。

### 5.3 平仓

作为用户，我希望随时平仓并领取链上可兑现权益。

验收标准：

- 平仓前必须先使用最新有效价格完成重置或微型结算。
- 平仓确认页展示预估可提取金额。
- 平仓后仓位状态变为 closed，不能重复提款。

### 5.4 市场选择

作为用户，我希望选择不同杠杆市场，并理解它们是相互隔离的。

验收标准：

- 每个市场展示标的、杠杆、抵押资产、总本金、多空比例、最近重置价和当前兑现率。
- UI 明确提示“不同杠杆市场不共享流动性”。

## 6. 功能需求

### 6.1 钱包连接

- 支持 Solana 主流钱包。
- 显示 USDC 余额和 SOL gas 余额。
- 钱包断开后隐藏敏感操作。

### 6.2 市场列表

字段：

- 标的。
- 杠杆。
- 抵押资产。
- 多头总本金。
- 空头总本金。
- 多头总名义价值。
- 空头总名义价值。
- 当前预言机价格。
- 最近重置价格。
- 市场状态。

### 6.3 开仓表单

字段：

- 标的市场。
- 方向：多 / 空。
- 保证金金额。
- 杠杆倍数。
- 手续费。
- 名义价值。
- 最大亏损。
- 预估交易后市场多空占比。

校验：

- 金额大于最小保证金。
- 金额不超过用户余额。
- 市场未暂停。
- 预言机价格有效。
- 单仓规模和市场总规模未超过限制。

### 6.4 仓位详情

字段：

- 仓位 ID。
- 开仓价。
- 最近重置价。
- 当前预言机价。
- 方向。
- 杠杆。
- 本金。
- 已锁定损失。
- 剩余本金。
- 参考盈亏。
- 可提取权益估算。
- 盈利兑现率。
- 创建时间。
- 最近结算时间。

### 6.5 平仓流程

- 拉取最新价格。
- 预估平仓金额。
- 用户确认风险。
- 发起平仓交易。
- 交易完成后更新仓位列表。

### 6.6 风险教育

必须包含：

- 永不爆仓不等于永不亏损。
- 最大亏损为本金。
- 参考盈利不等于可提取盈利。
- 低流动性市场中盈利可能无法全额兑现。
- 即使当前权益为 0，仓位仍保留方向性敞口；后续价格反转并出现新的对手盘亏损时，仓位可恢复部分或全部可提取权益。
- “不设置 Depleted 状态”的产品含义是：仓位不会因为当前权益归零而被系统判定死亡。它暂时没有可提取金额，但仍是一张有效仓位凭证。

## 7. 非功能需求

### 7.1 安全性

- 所有资金移动必须由合约约束。
- 所有价格必须经过有效性检查。
- 平仓和提款必须防止重复执行。

### 7.2 性能

- 常规开仓和平仓在 Solana 正常拥堵下应可被普通用户接受。
- 重置算法不得要求单笔交易遍历无限数量仓位。真实资金合约必须使用 settlement epoch、分片批处理和仓位懒结算。
- 当市场处于 epoch 结算中时，前端必须展示结算进度，并明确哪些仓位估算值尚未应用最新最终化 epoch。

### 7.3 可解释性

- 每个用户资金变化应能追溯到价格、市场状态和公式。
- 前端应提供交易后解释，而不是只展示结果。

### 7.4 可观测性

- 所有关键指令 emit 事件。
- 提供索引服务统计市场状态、重置历史和用户仓位。

## 8. 页面规划

### 8.1 首页

- 产品口号。
- 永不爆仓解释。
- 风险上限说明。
- 当前支持市场。

### 8.2 交易页

- 市场选择。
- K 线或价格图。
- 开仓面板。
- 市场统计。
- 用户仓位列表。

### 8.3 仓位页

- 当前仓位。
- 历史仓位。
- 每个仓位的结算明细。

### 8.4 风险模拟器

- 用户输入方向、杠杆、本金、价格路径。
- 展示在不同对手盘流动性下的权益变化。
- 用于教育用户理解兑现率和锁定损失。

### 8.5 协议统计页

- TVL。
- 成交量。
- 手续费。
- 市场多空分布。
- 平均盈利兑现率。
- 预言机状态。

## 9. 数据指标

- 日活交易钱包数。
- 新开仓数量。
- 平仓数量。
- 市场 TVL。
- 总名义未平仓量。
- 平均仓位持续时间。
- 盈利兑现率分布。
- 用户亏损超过本金事件数，目标为 0。
- 协议资不抵债事件数，目标为 0。

## 10. 上线门槛

测试网上线前必须满足：

- 核心合约单元测试通过。
- 清算引擎属性测试覆盖主要不变量。
- 本地仿真可复现白皮书关键案例。
- 前端明确展示参考盈亏与当前权益。
- 所有风险提示通过产品审查。

主网上线前必须额外满足：

- 外部安全审计完成。
- 经济攻击仿真完成。
- 设置市场规模上限。
- 设置紧急暂停权限和多签。
- 完成法律风险初评。
