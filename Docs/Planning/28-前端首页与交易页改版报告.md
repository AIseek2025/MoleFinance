## 2. 首页（落地页）

### 2.1 设计与信息架构

- 文件：`frontend/src/landing/LandingPage.tsx` + `frontend/src/landing/landing.css`
- 全部类名以 `lp-` 前缀隔离，零侵入既有 `styles.css`。
- 暗色 + 渐变光晕 + 网格背景，粘性导航栏滚动变色，响应式（≤720px 折叠）。

页面分区：

1. **Hero**：主标题"永不爆仓的链上杠杆交易" + 渐变副标题；实时价格条（现价 / 强平价=`无·None` / 最大亏损=`本金+费用` / 结算频率=`每区块`）。现价取自 `FeedSnapshot.indexer.market.midPriceMicro`。
2. **机制卡片（6 个）**：永不爆仓、对手盘即清算池、归零可复活、杠杆分场、当前区块清算、预言机封套保护。
3. **对比表**：传统永续合约（✕ 强平 / 插针 / 罚金 / 资金费率 / 归零出局）vs MoleOption（✓ 无强平 / 逐块清算 / 最大亏损=本金+费 / 零和透明 / 可复活）。
4. **四步流程**：选择市场 → 进入子池铸造 shares → Keeper 逐块清算 → 随时平仓提取。
5. **诚实风险提示** + 终版 CTA + 页脚。

### 2.2 机制文案来源（与协议一致）

所有"永不爆仓"相关表述严格对齐 `Docs/Planning/08-杠杆交易场与风控设计.md`：

- "永不爆仓"= 仓位不会被**传统强平**，**不表示不会亏完本金**；最大损失为本金与相关费用。
- 显示盈利为参考值，实际可提取金额取决于对手盘亏损。
- 当前权益为 0 不代表仓位被删除；价格反转且出现新对手盘亏损时仓位可重获权益。

> 文案没有夸大，把风险写在了首页与下单区，避免误导。

## 3. 交易页（Hyperliquid 风格交易终端）

### 3.1 布局

- 文件：`frontend/src/trade/TradeView.tsx`、`frontend/src/trade/PriceChart.tsx`、`frontend/src/trade/trade.css`
- 三列栅格（桌面）：`图表 1fr | 多空盘口 248px | 下单区 320px`；底部持仓/历史标签横跨左列。
- 响应式：≤1100px 隐藏盘口列，≤760px 单列堆叠。

### 3.2 K 线接入

- 引入依赖 `lightweight-charts@^4.2.0`（TradingView 开源图表库）。
- devnet 无历史 OHLC，故策略为：首帧用围绕现价的随机游走**合成回填**约 80 根蜡烛，之后把预言机逐块喂价**实时折叠**进当前蜡烛（`series.update`），按所选周期分桶。
- 周期切换：`5s / 15s / 1m / 5m`。
- 价格源即 `market.midPriceMicro`（由 keeper 每 ~6s 推送 `set_price + sync_pool` 驱动）。

### 3.3 下单功能区

- 做多 / 做空切换、市价单（"按预言机价格逐块结算"）、保证金（USDC）输入 + 快捷金额（100/500/1k/5k）、子池选择。
- 汇总：预计开仓价、价格封套（±0.50%）、开仓费（0.05%）、最大亏损（本金+费）、**强平价=无**。
- 下单链路**复用现有 wasm 指令编码器与钱包适配器**，与原 `TraderPanel` 完全一致：
  - `buildOpenPositionTx` / `buildClosePositionTx`（`tx/wasmBuilder.ts`）
  - `wallet.signAndSubmit({ description, borshBytes })`
  - `buildEnvelope` 价格封套（±50 bps）、`nextPositionId` 防抖位移
- 未引入任何新的链上交互路径，安全面不变。

### 3.4 多空盘口（替代订单簿）

MoleOption 是池模型、无订单簿，故用**真实多空本金分布**替代：多/空本金占比条、净偏斜、多空仓位数、待兑付 recovery。数据来自 `feed/openInterest.ts` 聚合，不伪造盘口。

### 3.5 持仓与交易历史

- **持仓表**：方向 / 账户 / 子池 / 数量 / 保证金 / 最大亏损 / 开仓时间 / 平仓按钮，数据来自 `feed.positions`。
- **交易历史**：本会话提交记录（开/平/失败，含签名缩写），如实标注为会话内日志。
