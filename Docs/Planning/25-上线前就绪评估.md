## 5. 本轮（wave 28）落地范围

聚焦沙箱内、可测、launch-relevant 的「协议总览」闭环（前后端）：

1. **后端** `ops_toolkit::protocol_summary`：`MultiMarketHealthReport` → `ProtocolSummary`（市场总数、按 overall 状态计数、firing 检查总数、worst exit、最高 firing 严重度）+ JSON 渲染，供状态页/告警单行消费。
2. **前端** `feed/protocolStats.ts` + `OverviewPanel`：从 live `MultiMarketView` 聚合协议级 TVL（Σ 子池抵押）、未平仓（数量+抵押）、多空偏斜、每市场健康/对账，作为**默认落地页**。

## 6. 上线前剩余（沙箱外，wave 29+ 路线）

1. 部署期：真实 program ID + IDL 发布 + devnet 集成矩阵。
2. solana-client 版 `RpcAccountSource` shim → prober live base fetcher 端到端真集群。
3. 真实浏览器钱包适配（Phantom/Backpack），真签名/提交。
4. 持仓历史 / 已平仓 PnL 页（依赖事件回放）。
5. 外部安全审计 firm 启动 + 修复闭环。
