## 1. 项目概述

MoleOption 是一套链上清算引擎 + 撮合/分配模型，核心是 `clearing_core` 纯函数清算引擎，
通过 Anchor 程序 `mole-option` 落地为链上指令，配套 keeper（同步预言机价格、分配 dormant 收益）、
indexer（链上状态镜像）、ops-toolkit（健康检查/协议聚合）与 React 前端。

- 语言/框架：Rust（引擎与链上程序）、Anchor 0.31、TypeScript + React + Vite（前端）。
- 当前状态：**已部署到 Solana devnet 并端到端跑通**（喂价 → sync_pool → 前端实时价格）。
- 未上主网。主网前置项见 §9。

## 4. 测试网（devnet）部署步骤（从零复现）

> 前提：本机已装 Rust + rustup、Solana CLI、Node 18+。所有 `node` 脚本已内置
> `dns.setDefaultResultOrder("ipv4first")`，规避无 IPv6 出口环境下 undici fetch 卡 10s 的问题。

### 4.1 安装 SBF 工具链

```bash
scripts/install-platform-tools.sh        # 健壮下载 platform-tools（断点续传）
# 确认：cargo-build-sbf 可用、~/.cache/solana/<ver>/platform-tools 存在
```

> 构建 SBF 时需让 rustup 的 cargo shim 在 PATH 前（识别 `+toolchain`）：
> `export PATH="$HOME/.cargo/bin:$HOME/.local/share/solana/install/active_release/bin:$PATH"`

### 4.2 资金

```bash
solana config set --url devnet --keypair ~/.config/solana/id.json
# 领测试币：公共 faucet 常限流，建议用 https://faucet.solana.com 手动领 ≥6 SOL
solana balance
```

### 4.3 部署主程序 mole-option

```bash
export SOLANA_RPC_URL="https://devnet.helius-rpc.com/?api-key=YOUR_KEY"  # 或公共 RPC
scripts/deploy-devnet.sh build      # 剥离 yanked dev-deps 后 cargo build-sbf
scripts/deploy-devnet.sh deploy     # solana program deploy（首次=新建，同 id 重跑=升级）
```

大程序（~680KB ≈ 600 笔写交易）部署/升级注意：
- 免费 RPC 易限流卡死，并把 ~5 SOL 锁进 write-buffer。**用持久 buffer keypair 续传**：
  `solana program deploy <so> --program-id <prog-kp> --buffer <持久 buffer-kp> --use-rpc ...`
  中途卡死 kill 后重跑会**续写同一 buffer**，不重复烧钱。
- 卡死/失败后回收 buffer 资金（公共 RPC 即可）：
  `solana program close --buffers -k ~/.config/solana/id.json -u https://api.devnet.solana.com`

### 4.4 部署 mock-oracle（devnet 才需要）

```bash
cd programs/mock-oracle
export PATH="$HOME/.cargo/bin:$HOME/.local/share/solana/install/active_release/bin:$PATH"
cargo-build-sbf
solana program deploy target/deploy/mock_oracle.so \
  --program-id target/deploy/mock_oracle-keypair.json --use-rpc
```

### 4.5 初始化协议状态 + 拉起 keeper

```bash
cd frontend
export SOLANA_RPC_URL="https://api.devnet.solana.com"
npm run keeper:setup    # 幂等：建价格账户 + SOL-PERP 市场 + sub_pool + long/short ledger
npm run keeper          # 循环：每 6s set_price + sync_pool（前台）
```

`keeper:setup` 会打印 `VITE_MARKET_PDA`，回填到 `frontend/.env.local`。

### 4.6 前端

```bash
cd frontend
cp .env.example .env.local   # 按需填 VITE_RPC_URL（生产建议用专用 RPC key）
npm install && npm run dev
# 浏览器：http://localhost:5173/?feed=live  （?feed=live 才切到链上实时数据）
```

## 5. 前后端 / 合约开发说明

### 5.1 合约（mole-option）

- 账户/指令定义在 `programs/mole-option/src/instructions/`，状态在 `state.rs`。
- 关键指令：`initialize_global_config` / `initialize_market` / `initialize_sub_pool` /
  `initialize_distribution_ledger` / `sync_pool` / `open_position` / `close_position` /
  `claim_dormant_recovery` / `pre_sync_dormant_bucket` / 治理类（pause/freeze/...）/ keeper-leader 锁。
- `sync_pool` 账户契约：`sub_pool, market, long_ledger, short_ledger, oracle_price_feed, clock`，
  remaining_accounts 传 long/short 活跃 bucket（数量由参数 `long/short_bucket_count` 指定）。
- 预言机校验走 `pyth-adapter::validate_price_account`：要求账户 owner == `market.oracle_program_id`，
  Pythnet v2 布局，status=trading，`current_slot - pub_slot ≤ max_oracle_age_slots`，置信度达标。

### 5.2 引擎（clearing-core）

- 价格刻度 `PRICE_SCALE = 1e8`（即报告价 ×1e8 存 u64）。
- **本次关键修复（见 §6）**：首价种入。

### 5.3 后端 keeper / ops

- keeper 职责：同步预言机价（sync_pool）、lazy 模式下 `pre_sync_dormant_bucket` 排空 ledger、
  回收死 bucket。`keeper-leader` 锁用于多实例选主（避免重复 sync）。
- ops-toolkit `protocol_summary` 输出协议级聚合（TVL/持仓/健康），prober 暴露 Prometheus 指标。

### 5.4 前端

- live 模式经 `?feed=live` 切到 `WebSocketFeedAdapter`，用 `keeper-decoder` 的 wasm 解码链上账户。
- Vite `server.fs.allow:['..']` 已放开，使 dev server 能加载仓库外的 `crates/keeper-decoder/pkg/*.wasm`。
- 环境变量经 `import.meta.env`（`VITE_RPC_URL` / `VITE_MOLE_PROGRAM_ID` / `VITE_MARKET_PDA`），
  Vite 构建时内联 —— 故**生产构建产物会暴露 RPC key**，务必用域名受限/限流的 key。

## 6. 本次关键修复：首价种入（critical）

**问题**：全新 `SubPool` 初始化时 `last_price = 0`；而 `clearing_core::sync_pool` 在
`last_price == 0` 时执行 `price_move_bps(0, p_now)` 会 `DivByZero` 报错；`open_position`
内部也是先调 `sync_pool`。结果：**任何市场的第一次 sync、第一笔开仓都会失败**——首价无种入路径。

**修复**：在 `sync_pool` 增加首价种入分支——当 `last_price == 0`（此时池子必然无持仓/无权益，
无可分配项）时，直接记录首价并返回。所有调用方（sync/open/close/claim）受益。

- 单测：`crates/clearing-core/src/tests.rs` 新增 `fresh_pool_first_sync_seeds_price`、
  `fresh_pool_first_open_succeeds`，`cargo test -p clearing-core` 全绿。
- 已重建并**升级部署**到 devnet（同 program id，Data Length 692648）。

## 7. 运维 Runbook（devnet）

### 7.1 keeper 常驻服务（macOS launchd）

已安装 LaunchAgent：`~/Library/LaunchAgents/com.moleoption.keeper.plist`
（`RunAtLoad` 登录自启 + `KeepAlive` 崩溃自重启，日志 `/tmp/mole/keeper.log`，包装脚本
`frontend/scripts/keeper-daemon.sh`）。

```bash
tail -f /tmp/mole/keeper.log                                       # 看实时 tick
launchctl print    gui/$(id -u)/com.moleoption.keeper | grep -E 'state|pid'
launchctl kickstart -k gui/$(id -u)/com.moleoption.keeper          # 重启
launchctl bootout  gui/$(id -u)/com.moleoption.keeper              # 停 + 取消自启
# 彻底移除：bootout 后 rm ~/Library/LaunchAgents/com.moleoption.keeper.plist
```

成本：devnet 每笔 ~0.000005 SOL，约 0.07 SOL/天，可忽略。

### 7.2 常见故障与处置（本次实战踩坑）

| 现象 | 原因 | 处置 |
|---|---|---|
| `program deploy` 卡死、0% CPU、~5 SOL 消失 | 免费 RPC 限流，SOL 锁进 write-buffer | 公共 RPC `program close --buffers` 全额回收；改用持久 buffer keypair 续传 |
| `error sending request` 频发 | Helius 免费档被突发流量限流 | 轻调用/finalize 改走公共 devnet RPC；或等冷却 |
| node 脚本 `fetch failed`（~10s 超时），但 curl 正常 | undici Happy-Eyeballs 卡 IPv6（沙箱无 IPv6 出口） | 脚本顶部 `dns.setDefaultResultOrder("ipv4first")`（已内置） |
| `cargo build-sbf`：`no such command +<toolchain>` | PATH 上 Homebrew cargo 在前，不识别 `+toolchain` | 把 `~/.cargo/bin`（rustup shim）置于 PATH 最前 |
| `cargo build-sbf`：solana_rbpf 0.8.1 yanked | dev-deps 拉到 yanked 版本 | 构建时临时剥离 `[dev-dependencies]`（`deploy-devnet.sh` 已自动处理并还原） |
| keeper sync 报 `OracleValidationFailed` | 价格账户过期/不存在 | 确认 keeper 在跑（喂价盖当前 slot）；价格账户=`uhTAD…`，owner 必须=mock 程序 |

3. **资金账户**：创建并校验 `vault` / `fee_vault` 的 SPL token 账户与真实抵押物 mint（当前为占位 mint，
